01 / 题面

赛题任务

四个不同的 Tron2 双臂任务,难度递增、全程 easy 环境;每段为专家一次成功操作的真实回放,头部视角 + 正面视角双机位(同一次操作、逐帧对齐)。

项说明
模型ACT(Action Chunking Transformer,~80M,PyTorch,RoboTwin自带实现)
自由度阶梯任务难度递增,每门累积解锁一项新自由度:T1 直接跑 → T2 自采数据 → T3 +自写 config → T4 全放开。
数据T1 下发处理好的数据;T2–T4 选手自采 —— 主办方下发可改专家 + 采集管线,选手跑"采集 → process_data → 训练"。
02 / 赛制

T1–T4 四任务,顺序解锁

四个任务难度递增,每门累积解锁一项新自由度(跑通→自采数据→自写 config→全放开)。前档达标才能交后档;各档各交各的模型、各自评测。前三题通关后进入 T4 比拼最终得分,T1/T2/T3 按成功率阈值判达标解锁。

题号 任务 主攻技能 提交物 计分
T1adjust_bottle
单臂抓取,极易		工具链入门:给处理好数据 + config 跑通 ACT(不自采)仅权重分级 SR ≥ τ₁(松)→ 解锁
T2lift_pot
双臂举锅		数据采集:用可改专家自采(采集量 / 域随机化 / 场景选择 / 改进专家)权重 + 代码分级 SR ≥ τ₂(须过 T1)
T3stack_bowls_two
双臂叠 2 碗		训练配置:自采数据 + 自己写训练 config(LR / 步数 / chunk / temporal-agg)权重 + 代码分级 SR ≥ τ₃(须过 T2)
T4stack_bowls_three
双臂叠 3 碗 · 主榜		全放开:数据自采 / 网络改造 / 推理闭环任意发挥,训更好的策略权重 + 代码T4 得分，满分100(须过 T3)
T1–T3 达标条件: 成功率SR ≥ τ　达标即解锁,本身不计分
T4排行榜分数: T4 得分 /100 降序　三个碗摆对位置各得 1/3、叠满得分 100
03 / 评测

计分:线上 70% + 完赛 30%

总评 = 线上排行榜 70% + 完赛材料 30%(完赛材料见 §04 最终交付物)。线上只有 T4 计分,用末态分层得分而非二值成功率——三个碗叠到第几层都有分,区分度比"成/败"更细。

总评构成占比说明
线上排行榜70%自动评测:主榜按 T4 得分 /100 降序;T1–T3 为顺序解锁门(pass/fail),达标才能刷 T4,本身不计分。
完赛材料30%赛末提交 开源代码包 + ≤3 分钟讲解 / 演示视频,由评委评审(评分细则赛前公布)。

线上比分:T4 得分判据

T4 得分 /100判据
100(满分)三个碗全部叠好,每层都落在锚点 C = [0, −0.1] 的 xy ±4cm 内。
66.7叠到第二层(两个碗稳定就位)。
33.3叠到第一层(第一个碗就位)。
0未完成第一层。

三层各占 1/3、逐层相加,叠满 = 100(不做平均)。队伍 T4 分 = 私有 100 个 seed episode 的得分均值,在 episode 末态单独算一次(不写进逐步调用的 check_success),由评测内核从末态位姿重算,选手自报无效。

评测机制

进程内 ACT评测为 in-process:加载选手 ACT 权重(torch)→ 同进程内跑 sim rollout → 末态算分。单卡、单一 RoboTwin 环境,无跨进程桥接。
取均值同一 policy 重复 rollout 取均值(EVAL_REPEATS 默认 5),消 ACT 采样噪声。
公开 / 私有 seed公开 seed 随脚手架下发供本地自测(本地分 = dev 榜分);私有 100 seed 只在主办方评测机,与公开互不重叠(防过拟合)。
选手代码生效T2/T3/T4 的代码包在评测进程内 shadow 官方 ACT —— 你自定义的数据 IO / 网络结构 / 推理闭环真实生效。
04 / 流程

从 clone 到上榜

clone 公开仓,一键装环境;本地自评跑的就是主办方评测的同一套 harness,只换种子(评测细节见上节)。

数据来源:T1 主办方下发处理好的数据;T2–T4 选手用下发的可改专家 + 采集管线自采。安装 / 自采 / 数据处理 / 训练 / 评测的 ACT 代码命令见 参赛文档 →。提交用 submit.py(唯一通道,带 token):

# T1:只交权重(turnkey 题,评测机跑官方 config) python submit/submit.py --track T1 --token <你的令牌> --ckpt-dir ./ckpt/T1 # T2 / T3 / T4:权重 + 整库代码包 python submit/submit.py --track T2 --token <你的令牌> --code-dir . --ckpt-dir ./ckpt/T2 python submit/submit.py --track T3 --token <你的令牌> --code-dir . --ckpt-dir ./ckpt/T3 python submit/submit.py --track T4 --token <你的令牌> --code-dir . --ckpt-dir ./ckpt/T4
机制说明
token 鉴权提交带 token,后端校验;每日 3 次正式提交(本地自测不限次)。无报名、无队名。
顺序解锁过 T1 才能交 T2,依此类推到 T4;T1/T2/T3 过了即锁定。
安全解包做 zip-slip / symlink 防护;ckpt 备案 SHA256;评测子进程降权 + 沙箱无网络出口,选手代码读不到私有 seed。

最终交付物

开源代码 + 文档完整代码开源到 GitHub,附技术设计文档,须可复现。
演示视频≤ 3 分钟,屏幕录制 + 画中画(摄像头):本人出镜讲解,现场 Demo 运行。
05 / 榜单

排行榜(匿名)

匿名榜:每 token 取最佳分,只显示 token 尾 6 位。T1–T3 显示达标门(✓ 达标 / ○ 未达标 / · 未提交)。主榜次序:有 T4 成绩者在前(T4 得分降序),其后无 T4 者按答题进度(过 T3 > 过 T2 > 过 T1)。每 60s 刷新。

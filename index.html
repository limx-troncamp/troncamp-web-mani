01 / 题面

赛题任务

四个不同的 TRON 2 双臂任务,难度递增、全程 easy 环境;每段为专家一次成功操作的真实回放,头部视角 + 正面视角双机位(同一次操作、逐帧对齐)。

项说明
模型ACT(Action Chunking Transformer,~80M,PyTorch,RoboTwin自带实现)
难度由易到难:T1 单臂调瓶 → T2 双臂抓举 → T3 叠 2 碗 → T4 叠 3 碗;全程 easy 环境,不叠加额外扰动,难度只来自任务本身。
数据四题全部自采,不下发成品数据集。主办方只给两样:① 调好的专家策略(能采出成功轨迹)、② 现成的采集脚本(一条命令就能跑)。选手用它们跑完整三步 —— 采集(专家自动演示,存成轨迹)→ process_data(转成 ACT 训练格式)→ 训练(得到策略权重)。区别只在训练 config:T1 连 config 也一起给(turnkey,直接照跑),T2 起训练 config 自己写。
02 / 赛制

T1–T4 四任务,顺序解锁

四个任务由易到难,共用同一条「自采数据 → 训练 → 评测」流水线:T1 照跑给定的 turnkey 训练 config,T2 起自己写 config。顺序解锁:前档达标(SR 过阈值)才能提交后档,各档独立提交模型、独立评测。T1/T2/T3 只判达标不计分,通关后进入 T4 比拼最终得分。

题号 任务 主攻技能 提交物 计分
T1adjust_bottle
单臂抓取,极易		入门:用下发的专家自采 + 给定 turnkey 训练 config,照跑训练 ACT仅权重SR ≥ τ₁ = 0.50 → 解锁
T2grab_roller
双臂抓擀面杖		用下发的专家自采 + 自己写训练 config权重 + 代码SR ≥ τ₂ = 0.60(须过 T1)
T3stack_bowls_two
双臂叠 2 碗		用下发的专家自采 + 自己写训练 config权重 + 代码SR ≥ τ₃ = 0.50(须过 T2)
T4stack_bowls_three
双臂叠 3 碗 · 主榜		用下发的专家自采 + 自己写训练 config,训更好的策略权重 + 代码T4 得分,满分 100(须过 T3)
T2 / T3 / T4 主攻技能(自由发挥,不设上限):① 基于下发的专家策略自行提高 → ② 自行做数据采集与处理 → ③ 数据增强 → ④ 自己实现网络结构进行训练。T1 只需跑通给定 turnkey config;T2 起以上全部放开。
T1–T3 达标条件: 成功率 SR ≥ τ　(τ₁ = 0.50 / τ₂ = 0.60 / τ₃ = 0.50)　达标即解锁,本身不计分
T4排行榜分数: T4 得分 /100 降序　三个碗摆对位置各得 1/3、叠满得分 100
03 / 评测

计分:线上 70% + 完赛 30%

总评 = 线上排行榜 70% + 完赛材料 30%(完赛材料见 §04 最终交付物)。线上只有 T4 计分,用末态分层得分而非二值成功率——三个碗叠到第几层都有分,区分度比"成/败"更细。

总评构成占比说明
线上排行榜70%自动评测:主榜按 T4 得分 /100 降序;T1–T3 为顺序解锁门(pass/fail),达标才能刷 T4,本身不计分。
完赛材料30%赛末提交 开源代码包 + ≤3 分钟讲解 / 演示视频,由评委评审。

线上比分:T4 得分判据

T4 得分 /100判据
100(满分)三个碗全部叠好,每层都落在锚点 C = [0, −0.1] 的 xy ±4cm 内。
66.7叠到第二层(两个碗稳定就位)。
33.3叠到第一层(第一个碗就位)。
0未完成第一层。

三层各占 1/3、逐层相加,叠满 = 100(不做平均)。队伍 T4 分 = 私有 100 个 seed episode 的得分均值,在 episode 末态单独算一次(不写进逐步调用的 check_success),由评测内核从末态位姿重算,选手自报无效。

评测机制

进程内 ACT评测为 in-process:加载选手 ACT 权重(torch)→ 同进程内跑 sim rollout → 末态算分。单卡、单一 RoboTwin 环境,无跨进程桥接。
取均值同一 policy 重复 rollout 取均值(EVAL_REPEATS 默认 5),消 ACT 采样噪声。
公开 / 私有 seed公开 seed 随脚手架下发供本地自测(与官方评测同内核、同分布,只换种子;--repeats 5 可逼近官方口径);官方榜用私有 100 seed,只在主办方评测机、与公开互不重叠(防过拟合)。
选手代码生效T2/T3/T4 的代码包在评测进程内 shadow 官方 ACT —— 你自己写的训练 config / 数据处理真实生效。
04 / 流程

从 clone 到上榜

clone 公开仓,一键装环境;本地自评跑的就是主办方评测的同一套 harness,只换种子(评测细节见上节)。

数据:四题全部自采 —— 主办方下发调好的专家 + 现成的采集脚本,选手跑"采集 → process_data → 训练"(T1 额外给 turnkey config)。安装 / 采集 / 数据处理 / 训练 / 评测的 ACT 代码命令见 参赛文档 →。提交用 submit.py(唯一通道,带 token):

# T1:只交权重(turnkey 题,评测机跑官方 config) python submit/submit.py --server <主办方公布地址> --track T1 --token <你的令牌> --ckpt external/robotwin_local/policy/ACT/act_ckpt/act-adjust_bottle/<config>-<num>/policy_best.ckpt # T2 / T3 / T4:权重 + 代码包。--code-dir 指含 policy/ 的 RoboTwin 根(评测遮蔽 <root>/policy;自动只打 policy/,~2MB) python submit/submit.py --server <主办方公布地址> --track T2 --token <你的令牌> --code-dir external/robotwin_local --ckpt external/robotwin_local/policy/ACT/act_ckpt/act-grab_roller/<config>-<num>/policy_best.ckpt python submit/submit.py --server <主办方公布地址> --track T3 --token <你的令牌> --code-dir external/robotwin_local --ckpt external/robotwin_local/policy/ACT/act_ckpt/act-stack_bowls_two/<config>-<num>/policy_best.ckpt python submit/submit.py --server <主办方公布地址> --track T4 --token <你的令牌> --code-dir external/robotwin_local --ckpt external/robotwin_local/policy/ACT/act_ckpt/act-stack_bowls_three/<config>-<num>/policy_best.ckpt

--ckpt 直接指向你要交的那个 checkpoint 文件;CLI 自动附带同目录的 dataset_stats.pkl、并作为评测加载名 policy_last.ckpt 上传——交哪个就评哪个,无需手动改名。训练会同时存 policy_best.ckpt(验证集最优)与 policy_last.ckpt(末轮);建议交 policy_best.ckpt(早停,实测显著高于末轮)。

机制说明
token 鉴权提交带 token,后端校验;每日 3 次正式提交(本地自测不限次)。无报名、无队名。
顺序解锁过 T1 才能交 T2,依此类推到 T4;T1/T2/T3 过了即锁定。
安全解包做 zip-slip / symlink 防护;ckpt 备案 SHA256;评测子进程降权 + 沙箱无网络出口(私有 seed 不出评测机);赛末对头部成绩受控复评后定名次。

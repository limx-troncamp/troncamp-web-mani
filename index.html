01 / 题面

四个任务与模型

四个不同的 Tron2 双臂任务,难度递增、全程 easy 环境;每段为专家一次成功操作的真实回放,头部视角 + 正面视角双机位(同一次操作、逐帧对齐)。

项说明
模型ACT(Action Chunking Transformer,~80M,PyTorch,RoboTwin 原生动作分块策略)。单张 4090 级显卡 <1h 量级可训完一轮,选手可日迭代。
难度轴任务本身难度递增(瓶 → 锅 → 2 碗 → 3 碗,Aloha 参考 SR 97 / 88 / 82 / 48);每个门再给一份"卡在阈值线下方"的次优起点物料(混质量数据池 / 欠调 config)逼出该门技能。全程 easy 环境、单 embodiment Tron2。
数据主办方下发 ACT-ready 处理好的数据(process_data 产),选手不自采;T2 给更大的混质量池(数据工程在其上做),T3/T4 给干净集。
02 / 赛制

T1–T4 四任务,顺序解锁

四个任务难度递增,每门主攻一项工程技能。前档达标才能交后档;各档各交各的模型、各自评测。只有 T4 计分(主榜),T1/T2/T3 是 pass/fail 门(过即锁)。

档 任务 主攻技能 · 次优起点 提交物 计分
T1adjust_bottle
单臂抓取,极易		工具链入门:给处理好数据 + turnkey config 跑通 ACT仅权重分级 SR ≥ τ₁(松)→ 解锁
T2lift_pot
双臂举锅		数据工程:在混质量大池上做筛选 / 配比 / load 时增广权重 + 代码分级 SR ≥ τ₂(须过 T1)
T3stack_bowls_two
双臂叠 2 碗		训练工程:在故意欠调的 config 上调 LR / 步数 / chunk / temporal-agg权重 + 代码分级 SR ≥ τ₃(须过 T2)
T4stack_bowls_three
双臂叠 3 碗 · 主榜		网络 / 闭环全开(架构改造 + 推理时闭环,不含 DAgger 自采)权重 + 代码分级分(graded)降序排名(主榜)(须过 T3)
T1–T3 门 = 分级 SR ≥ τ　达标即解锁,本身不计分
主榜(T4)= 分级分 graded 降序　末态 /3,无成功率/力质量/其它分量

阈值 τ₁/τ₂/τ₃ 由主办方赛前实测四任务 easy 基线后标定(T1≤T2≤T3),不写死单点值;EVAL_REPEATS 取均值消 ACT 采样噪声(默认 5)。

03 / 评测

主榜命根子:T4 分级分

只有 T4 计分,用末态分级分而非二值成功率——三个碗叠到第几层都有分,区分度比"成/败"更细。

分级分 /3判据
1(满分)三个碗全部叠好,且每层都落在锚点 C = [0, −0.1] 的 xy ±4cm 内 —— 等价于二值成功。
⅔叠到第二层(两个碗稳定就位)。
⅓叠到第一层(第一个碗就位)。
0未完成第一层。

队伍 T4 分 = 私有 100 个 seed episode 的末态分级分均值。分级分在 episode 末态单独算一次(不写进逐步调用的 check_success),由评测内核 graded_score 从仿真末态位姿重算,选手自报无效。

评测机制

进程内 ACT评测为 in-process:加载选手 ACT 权重(torch)→ 同进程内跑 sim rollout → 末态算分。单卡、单一 RoboTwin 环境,无跨进程桥接。
取均值同一 policy 重复 rollout 取均值(EVAL_REPEATS 默认 5),消 ACT 采样噪声。
公开 / 私有 seed公开 seed 随脚手架下发供本地自测(本地分 = dev 榜分);私有 100 seed 只在主办方评测机,与公开互不重叠(防过拟合)。
选手代码生效T2/T3/T4 的代码包在评测进程内 shadow 官方 ACT —— 你自定义的数据 IO / 网络结构 / 推理闭环真实生效。
04 / 流程

从 clone 到上榜

clone 公开仓,一键装环境;本地自评跑的就是主办方评测的同一套 harness,只换种子。评测为 in-process:加载 ACT 权重(torch)→ 同进程内跑 rollout → 末态算分(T4 分级分 / T1–T3 分级 SR)。

本地自评

# 在公开 seed 上本地出分(= 主办方评测同一套内核,只换种子) python starter/eval_local.py --ckpt-dir ./ckpt --track T1 # 跑策略 + 渲 rollout 视频(操作过程清晰可见,核查行为) python starter/watch_rollout.py --ckpt-dir ./ckpt --seed 0 --out rollout.mp4

提交

# T1:只交权重(ACT ckpt 目录:policy_last.ckpt + dataset_stats.pkl) python submit/submit.py --track T1 --token <你的令牌> --ckpt-dir ./ckpt # T2 / T3 / T4:权重 + 整库代码(CLI 自动打包,自动算 SHA256) python submit/submit.py --track T4 --token <你的令牌> --code-dir ./mycode --ckpt-dir ./ckpt
机制说明
token 鉴权提交带 token,后端校验 + per-token 每日限额(DAILY_LIMIT)。无报名、无队名。
顺序解锁过 T1 才能交 T2,依此类推到 T4;T1/T2/T3 过了即锁定。
dev / finaldev 公开(自评 / live 榜);final 私有(赛末权威定格排名,解锁前显示「赛末公布」)。
安全解包做 zip-slip / symlink 防护;ckpt 备案 SHA256;评测子进程降权 + 沙箱无网络出口,选手代码读不到私有 seed。

最终交付物

开源代码 + 文档完整代码开源到 GitHub,附技术设计文档,须可复现。
演示视频≤ 3 分钟,屏幕录制 + 画中画(摄像头):本人出镜讲解,现场 Demo 运行。

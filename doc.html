参赛文档

选手包内 ACT(Action Chunking Transformer)的代码使用参考:采集 → 数据处理 → 训练 → 评测 → 配置。沿用 RoboTwin 第一方 policy/ACT 脚本,命令复制即用,单卡可跑。本页只覆盖 ACT;参赛规则见主页。

从安装开始 RoboTwin 官方文档 ↗ 排行榜

命令的执行目录分两处:采集(§02)、本地自评与提交(§05)在选手包根目录执行;数据处理(§03)、训练(§04)、eval.sh(§05)在 external/robotwin_local/policy/ACT 下执行——下文这三步用 ( cd external/robotwin_local/policy/ACT && bash … ) 子 shell 写法,可从根目录直接复制运行。任务参数三元组 <task_name> <task_config> <expert_data_num> 拼成数据键 sim-<task_name>-<task_config>-<expert_data_num>(见 §06 配置)。

01 / 安装

安装

选手包根目录一键脚本会创建 conda 环境,并把 RoboTwin / SAPIEN / Curobo 与 ACT 的全部依赖一次装好——无需任何手动 pip install。

# 选手包一键装环境(唯一步骤):conda 环境 + RoboTwin / SAPIEN / Curobo + ACT 依赖全部装好 bash setup/install.sh

setup/install.sh 已自动装好 ACT 运行所需的全部子依赖,不需要再手动跑任何 pip 命令;装完即可直接进入 §02 采集。

02 / 采集

数据采集 collect(四题全部自采)

四题都自采 —— 用下发的调好专家在 TRON 2 上跑 RoboTwin 采集,只留成功 episode。专家 = external/robotwin_local/envs/<task_name>.py 的 play_once(),可直接改进。采集集数 / 域随机化 / 场景在 task_config 里调。数据经 JPEG 压缩,~12M/集(不炸盘)。

# 在选手包根目录执行;先 bash setup/install.sh(解析路径占位,curobo 规划器才能起) # bash collect_data.sh <task_name> <task_config> <gpu_id> —— 打 TRON 2 自碰撞 patch 后采集 bash collect_data.sh adjust_bottle adjust_bottle_200ep 0 # 采完用 process_data 转 ACT 16-D(见下节);collect / process_data / train 用同一 config 名(adjust_bottle_200ep)串起来

只有 T1 的采集 config adjust_bottle_200ep.yml 随包下发(turnkey);评测用的 <task>_clean.yml 也随包下发但勿改。T2–T4 的采集 / 训练 config 不随包下发,自己建:把 external/robotwin_local/task_config/adjust_bottle_200ep.yml 拷贝改名成 <task>_200ep.yml,调 episode_num / 场景后,把三步命令里的 adjust_bottle 换成对应任务名即可。

想采更多 / 更优数据 → 改 task_config 的集数与域随机化,或直接改进 envs/<task>.py 的专家。改动随提交的代码包生效。

(可选)想直观查看采集:把 task_config/<config>.yml 的 render_freq 从 0 改成 >0(如 20)会弹出 SAPIEN 仿真窗口 —— 仅在种子搜索阶段显示(正式落盘阶段固定关闭),且需带显示器 / 图形环境的机器(无头服务器 / SSH 无 X 会报错)。评测(§05)不支持可视化。纯调试便利,不影响成绩。

03 / 数据处理

数据处理 process_data

把 RoboTwin 采集的轨迹转成 ACT 训练格式,产物落在 processed_data/sim-<task>/<config>-<num>/(RoboTwin 官方两级目录约定)。四题自采(上一步)后都用本步转 16-D 格式再训练(JPEG 压缩存储)。

# bash process_data.sh <task_name> <task_config> <expert_data_num> —— 在 external/robotwin_local/policy/ACT 下执行 ( cd external/robotwin_local/policy/ACT && bash process_data.sh adjust_bottle adjust_bottle_200ep 200 )
参数含义
task_name任务名,如 adjust_bottle / grab_roller / stack_bowls_two / stack_bowls_three
task_config配置名(数据采集设定),如 adjust_bottle_200ep
expert_data_num用于训练的专家轨迹数,如 200
04 / 训练

训练 train

训练脚本 train.sh 内部已设好 ACT 标准超参(kl_weight 10 / chunk_size 50 / hidden_dim 512 / dim_feedforward 3200 / batch_size 8 / lr 1e-5 / state_dim 16),默认 num_epochs=6000,单卡可训。ckpt 落在 act_ckpt/act-<task_name>/<task_config>-<expert_data_num>/。

# bash train.sh <task_name> <task_config> <expert_data_num> <seed> <gpu_id> —— 在 external/robotwin_local/policy/ACT 下执行 ( cd external/robotwin_local/policy/ACT && bash train.sh adjust_bottle adjust_bottle_200ep 200 0 0 ) # 产出 ckpt → act_ckpt/act-adjust_bottle/adjust_bottle_200ep-200/
题号训练命令
T1bash train.sh adjust_bottle adjust_bottle_200ep 200 0 0
T2bash train.sh grab_roller grab_roller_200ep 200 0 0
T3bash train.sh stack_bowls_two stack_bowls_two_200ep 200 0 0
T4bash train.sh stack_bowls_three stack_bowls_three_200ep 200 0 0

表中训练命令同上方示例,都在 external/robotwin_local/policy/ACT 下执行(从根目录跑就套页头的 ( cd … && bash … ) 子 shell 写法)。T2–T4 的 <task>_200ep 是你在 §02 自建的 config 名(拷 adjust_bottle_200ep.yml 改名而来),不随包下发;只有 T1 的 adjust_bottle_200ep 是 turnkey 下发的。

参数含义
seed随机种子,如 0
gpu_idGPU 设备号,如 0(脚本内 export CUDA_VISIBLE_DEVICES)

需改步数 / 超参时直接编辑 train.sh 或底层 imitate_episodes.py 的参数;T2/T3/T4 的工程改动随提交的代码包在评测进程内生效。

参考耗时(单题全流程)

步骤参考耗时条件
采集 collect(§02)~1 h 55 min单卡 4090;搜 ~500 seed 采够 200 条成功轨迹
数据处理 process_data(§03)~1.5 min200 条轨迹转 ACT 16-D 格式
训练 train(§04)~9 h单卡 4090;num_epochs=6000
单题全套~11 h采集 + 处理 + 训练合计(不含评测与调参迭代)

主办方实测参考(单卡 4090),实际因机器 / 配置 / 数据量而异。赛期约 12 天、每日 1 次正式提交(本地自测不限次),单题全套约 11 h——请据此规划机时与迭代轮次。

05 / 评测

评测与部署 eval

评测脚本 eval.sh 加载 deploy_policy.yml 配置(temporal_agg=true 推理时闭环),跑 sim rollout,结果(含视频)落在工程根的 eval_result/。ckpt_setting 选 act_ckpt 下的 ckpt 目录,可与评测环境 task_config 不同(跨设定泛化测试)。

# bash eval.sh <task_name> <task_config> <ckpt_setting> <expert_data_num> <seed> <gpu_id> —— 在 external/robotwin_local/policy/ACT 下执行 # 同设定评测(评测环境 = 训练设定) ( cd external/robotwin_local/policy/ACT && bash eval.sh adjust_bottle adjust_bottle_200ep adjust_bottle_200ep 200 0 0 ) # 跨设定评测(用官方评测同款 clean 设定测泛化) ( cd external/robotwin_local/policy/ACT && bash eval.sh adjust_bottle adjust_bottle_clean adjust_bottle_200ep 200 0 0 )
参数含义
task_config评测环境的配置(测哪种设定)
ckpt_settingckpt 的来源设定(选 act_ckpt/act-<task>/<ckpt_setting>-<num>/)
结果视频 + 指标 → 工程根 eval_result/

本地自评(榜单同套内核)

比赛自测用选手包的 starter/eval_local.py 包装:跑的就是官方评测同一套 harness,同内核、同分布、只换种子(官方用私有 seed,与本地同为 --repeats 1,本地分是官方分的估计,不保证逐分相等)。

# 在选手包根目录执行;--ckpt-dir 指向训练产出的 act_ckpt 目录 python starter/eval_local.py --track T1 --ckpt-dir external/robotwin_local/policy/ACT/act_ckpt/act-adjust_bottle/adjust_bottle_200ep-200

达标阈值(同一 easy/clean 官方评测环境的成功率口径):T1 SR ≥ 0.50 / T2 SR ≥ 0.60 / T3 SR ≥ 0.50 —— 达标即顺序解锁(T1→T2→T3→T4),T1–T3 本身不计分;T4 是主榜分数(末态 graded /100,三个碗各占 1/3)。提交用 submit/submit.py --server <主办方公布地址> --track <Tn> --token <你的令牌> --ckpt <policy_best.ckpt>(令牌也可用 env TRONCAMP_TOKEN 提供;T2–T4 另加 --code-dir external/robotwin_local 打整库代码包)。

06 / 配置

配置文件与任务键

SIM_TASK_CONFIGS.json 定义每个数据键(dataset_dir / num_episodes / episode_len / camera_names);训练 / 评测的 --task_name 即此处的键。deploy_policy.yml 为评测推理配置(temporal_agg 等)。

题号数据键(task_name)train.sh 三元组
T1sim-adjust_bottle-adjust_bottle_200ep-200adjust_bottle adjust_bottle_200ep 200
T2sim-grab_roller-grab_roller_200ep-200grab_roller grab_roller_200ep 200
T3sim-stack_bowls_two-stack_bowls_two_200ep-200stack_bowls_two stack_bowls_two_200ep 200
T4sim-stack_bowls_three-stack_bowls_three_200ep-200stack_bowls_three stack_bowls_three_200ep 200

表中 T2–T4 的 <task>_200ep 为你自建的 config 名(见 §02,拷 adjust_bottle_200ep.yml 改名);只有 T1 的 adjust_bottle_200ep 随包下发。

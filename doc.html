参赛文档

选手包内 ACT(Action Chunking Transformer)的代码使用参考:采集 → 数据处理 → 训练 → 评测 → 配置。沿用 RoboTwin 第一方 policy/ACT 脚本,命令复制即用,单卡可跑。本页只覆盖 ACT;参赛规则见主页。

从安装开始 RoboTwin 官方文档 ↗ 排行榜

下文命令均在 ACT 训练栈目录执行:cd external/robotwin_local/policy/ACT。任务参数三元组 <task_name> <task_config> <expert_data_num> 拼成数据键 sim-<task_name>-<task_config>-<expert_data_num>(见 §05 配置)。

01 / 安装

安装

选手包根目录一键脚本会创建 conda 环境并装好 ACT 全部依赖。若要单独装 ACT 子依赖(脚本已包含),命令如下。

# 选手包一键装环境(推荐):conda 环境 + RoboTwin / SAPIEN / Curobo + ACT 依赖 bash setup/install.sh
# 或:单独装 ACT 子依赖(在 policy/ACT 目录) cd external/robotwin_local/policy/ACT pip install pyquaternion pyyaml rospkg pexpect mujoco==2.3.7 dm_control==1.0.14 opencv-python matplotlib einops packaging h5py ipython cd detr && pip install -e . && cd ..
02 / 采集

数据采集 collect(四题全部自采)

四题都自采 —— 用下发的调好专家在 TRON 2 上跑 RoboTwin 采集,只留成功 episode。专家 = external/robotwin_local/envs/<task_name>.py 的 play_once(),可直接改进。采集集数 / 域随机化 / 场景在 task_config 里调。数据经 JPEG 压缩,~12M/集(不炸盘)。

# 先 bash setup/install.sh(解析路径占位,curobo 规划器才能起) # bash collect_data.sh <task_name> <task_config> <gpu_id> —— 打 TRON 2 自碰撞 patch 后采集 bash collect_data.sh grab_roller grab_roller_200ep 0 # 采完用 process_data 转 ACT 16-D(见下节);collect / process_data / train 用同一 config 名(grab_roller_200ep)串起来

想采更多 / 更优数据 → 改 task_config 的集数与域随机化,或直接改进 envs/<task>.py 的专家。改动随提交的代码包生效。

03 / 数据处理

数据处理 process_data

把 RoboTwin 采集的轨迹转成 ACT 训练格式,产物落在 processed_data/sim-<task>/<config>-<num>/(RoboTwin 官方两级目录约定)。四题自采(上一步)后都用本步转 16-D 格式再训练(JPEG 压缩存储)。

# bash process_data.sh <task_name> <task_config> <expert_data_num> bash process_data.sh adjust_bottle adjust_bottle_200ep 200
参数含义
task_name任务名,如 adjust_bottle / grab_roller / stack_bowls_two / stack_bowls_three
task_config配置名(数据采集设定),如 adjust_bottle_200ep
expert_data_num用于训练的专家轨迹数,如 200
04 / 训练

训练 train

训练脚本 train.sh 内部已设好 ACT 标准超参(kl_weight 10 / chunk_size 50 / hidden_dim 512 / dim_feedforward 3200 / batch_size 8 / lr 1e-5 / state_dim 16),默认 6000 步,单卡可训。ckpt 落在 act_ckpt/act-<task_name>/<task_config>-<expert_data_num>/。

# bash train.sh <task_name> <task_config> <expert_data_num> <seed> <gpu_id> bash train.sh adjust_bottle adjust_bottle_200ep 200 0 0 # 产出 ckpt → act_ckpt/act-adjust_bottle/adjust_bottle_200ep-200/
题号训练命令
T1bash train.sh adjust_bottle adjust_bottle_200ep 200 0 0
T2bash train.sh grab_roller grab_roller_200ep 200 0 0
T3bash train.sh stack_bowls_two stack_bowls_two_200ep 200 0 0
T4bash train.sh stack_bowls_three stack_bowls_three_200ep 200 0 0
参数含义
seed随机种子,如 0
gpu_idGPU 设备号,如 0(脚本内 export CUDA_VISIBLE_DEVICES)

需改步数 / 超参时直接编辑 train.sh 或底层 imitate_episodes.py 的参数;T2/T3/T4 的工程改动随提交的代码包在评测进程内生效。

05 / 评测

评测与部署 eval

评测脚本 eval.sh 加载 deploy_policy.yml 配置(temporal_agg=true 推理时闭环),跑 sim rollout,结果(含视频)落在工程根的 eval_result/。ckpt_setting 选 act_ckpt 下的 ckpt 目录,可与评测环境 task_config 不同(跨设定泛化测试)。

# bash eval.sh <task_name> <task_config> <ckpt_setting> <expert_data_num> <seed> <gpu_id> # 同设定评测(评测环境 = 训练设定) bash eval.sh adjust_bottle adjust_bottle_200ep adjust_bottle_200ep 200 0 0 # 跨设定评测(评测环境 ≠ 训练设定,测泛化) bash eval.sh adjust_bottle adjust_bottle_dr adjust_bottle_200ep 200 0 0
参数含义
task_config评测环境的配置(测哪种设定)
ckpt_settingckpt 的来源设定(选 act_ckpt/act-<task>/<ckpt_setting>-<num>/)
结果视频 + 指标 → 工程根 eval_result/

本地自评(榜单同套内核)

比赛自测用选手包的 starter/eval_local.py 包装:跑的就是官方评测同一套 harness,同内核、同分布、只换种子(官方用私有 seed + --repeats 5,本地分是官方分的无偏估计,不保证逐分相等)。

# --ckpt-dir 指向训练产出的 act_ckpt 目录 python starter/eval_local.py --track T1 --ckpt-dir external/robotwin_local/policy/ACT/act_ckpt/act-adjust_bottle/adjust_bottle_200ep-200

达标阈值(同一弱随机评测环境的成功率口径):T1 SR ≥ 0.50 / T2 SR ≥ 0.60 / T3 SR ≥ 0.50 —— 达标即顺序解锁(T1→T2→T3→T4),T1–T3 本身不计分;T4 是主榜分数(末态 graded /100,三个碗各占 1/3)。提交用 submit/submit.py --server <主办方公布地址> --track <Tn> --ckpt <policy_best.ckpt>(T2–T4 另加 --code-dir external/robotwin_local 打整库代码包)。

06 / 配置

配置文件与任务键

SIM_TASK_CONFIGS.json 定义每个数据键(dataset_dir / num_episodes / episode_len / camera_names);训练 / 评测的 --task_name 即此处的键。deploy_policy.yml 为评测推理配置(temporal_agg 等)。

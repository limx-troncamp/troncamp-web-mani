参赛文档

完整实操指南:装环境 → 下载主办方数据 → 训练 ACT → 本地自评 → 提交上榜。命令均为 RoboTwin / 选手包真实入口,复制即用。单张 4090 级显卡即可全流程跑通。

开始安装 参赛流程 排行榜
01 / 安装

环境安装

clone 选手包(含 Tron2 本体 + RoboTwin runtime + ACT 训练栈 + 评测内核),一键脚本装好 conda 环境与全部仿真依赖。后续所有命令在选手包根目录执行。

# 1. clone 选手包 git clone https://github.com/limx-troncamp/troncamp-mani cd troncamp-mani # 2. 一键装环境:创建 conda 环境 + RoboTwin / SAPIEN / Curobo 依赖(按需配代理) bash setup/install.sh
02 / 数据

数据(主办方下发)

四题的 ACT-ready 数据由主办方处理好、随赛下发,选手不自采。下载后解压到 ACT 训练栈的 processed_data/ 下,目录名即 SIM_TASK_CONFIGS.json 里的任务键。

# 数据放置路径(<任务键> = 下表"任务键"列) external/robotwin_local/policy/ACT/processed_data/<任务键>/
题号任务键说明
T1sim-adjust_bottle-adjust_bottle_200ep-200干净集,跑通官方 config 即可
T2sim-lift_pot-lift_pot_mixed300-300混质量大池 —— 数据工程:筛选 / 配比 / load 时增广
T3sim-stack_bowls_two-stack_bowls_two_200ep-200干净集 + 故意欠调的 config —— 训练工程
T4sim-stack_bowls_three-stack_bowls_three_200ep-200干净集,主榜任务
03 / 训练

训练 ACT

ACT 训练沿用 RoboTwin 第一方脚本 train.sh,参数顺序 <任务> <配置> <集数> <seed> <GPU>。脚本内部已设好 ACT 标准超参(kl_weight 10 / chunk_size 50 / hidden_dim 512 / lr 1e-5 / state_dim 16),单卡可训。

# 进入 ACT 训练栈 cd external/robotwin_local/policy/ACT # bash train.sh <任务> <配置> <集数> <seed> <GPU> —— 以 T1 为例,用 GPU 0 bash train.sh adjust_bottle adjust_bottle_200ep 200 0 0 # 产出 ckpt → act_ckpt/act-adjust_bottle/adjust_bottle_200ep-200/
题号训练命令(在 policy/ACT 目录下)
T1bash train.sh adjust_bottle adjust_bottle_200ep 200 0 0
T2bash train.sh lift_pot lift_pot_mixed300 300 0 0
T3bash train.sh stack_bowls_two stack_bowls_two_200ep 200 0 0
T4bash train.sh stack_bowls_three stack_bowls_three_200ep 200 0 0

T2 / T3 / T4 的工程发挥(数据筛选、调参、改网络 / 闭环)在选手包代码里改;随提交的代码包会在评测进程内 shadow 官方 ACT,真实生效。

04 / 评测

本地自评

本地自评跑的就是主办方评测的同一套 harness,只换种子 —— 本地分 = dev 榜分。公开 seed 随包下发,本地不限次。--ckpt-dir 指向训练产出的 ckpt 目录。

# 公开 seed 上本地出分(--ckpt-dir = 上一步 act_ckpt 下对应目录) python starter/eval_local.py --track T1 --ckpt-dir external/robotwin_local/policy/ACT/act_ckpt/act-adjust_bottle/adjust_bottle_200ep-200 # 跑策略 + 渲染 rollout 视频,核查操作行为 python starter/watch_rollout.py --ckpt-dir external/robotwin_local/policy/ACT/act_ckpt/act-adjust_bottle/adjust_bottle_200ep-200 --seed 0 --out rollout.mp4
05 / 提交

提交

submit.py 是唯一提交通道,带 token。T1 只交权重(turnkey 题,评测机跑官方 config);T2 / T3 / T4 交权重 + 整库代码包(--code-dir .),你的数据 IO / 网络 / 闭环改动随代码包生效。--ckpt-dir 指向对应题训练产出的 ckpt 目录。

# T1:只交权重 python submit/submit.py --track T1 --token <你的令牌> --ckpt-dir <T1 ckpt 目录> # T2 / T3 / T4:权重 + 整库代码包 python submit/submit.py --track T2 --token <你的令牌> --code-dir . --ckpt-dir <T2 ckpt 目录> python submit/submit.py --track T3 --token <你的令牌> --code-dir . --ckpt-dir <T3 ckpt 目录> python submit/submit.py --track T4 --token <你的令牌> --code-dir . --ckpt-dir <T4 ckpt 目录>
机制说明
token 鉴权提交带 token,后端校验;每日 3 次正式提交(本地自测不限次)。无报名、无队名。
顺序解锁过 T1 才能交 T2,依此类推到 T4;T1 / T2 / T3 过了即锁定。
评分主榜 = T4 得分 /100 降序;T1–T3 为达标门(SR ≥ τ),pass/fail 不计分。
安全解包做 zip-slip / symlink 防护;ckpt 备案 SHA256;评测子进程降权 + 沙箱无网络出口,选手代码读不到私有 seed。
06 / FAQ

常见问题

要几张卡?单张 4090 级显卡即可训练与本地评测。
能自己采数据吗?不能。四题数据均由主办方下发,选手不自采。T2 的"数据工程"是在下发的混质量池上做筛选 / 配比 / 增广。
本地分和榜单分一致吗?一致。本地自评 = 主办方评测同一套 harness,只换种子;公开 seed 本地分 = dev 榜分。私有 100 seed 只在评测机、与公开互不重叠。
T1 为什么不交代码?T1 是 turnkey 保底题,跑通官方 config 即可,评测机强制用官方 ACT 跑;T2 / T3 / T4 才交代码包。
每天能提交几次?正式提交每队每日 3 次;本地自测不限次。
最终要交什么?赛末提交开源代码包 + ≤3 分钟讲解 / 演示视频(屏幕录制 + 画中画出镜)。总评 = 线上排行榜 70% + 完赛材料 30%。
